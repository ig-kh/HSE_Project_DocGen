# Протокол сбора данных

## Синтетические данные
Нужно случайно нагенерировать JSON с разными текстами и правилами отправки сообщений, далее при помощи большой модели (возможно через API) с повышенной температурой генерации нагенерировать исходные тексты сообщений и метаданные (в частности мета-сервер тайм отправки).

## Внутренний Краудсорс
Потенциально на своих внутренних ресурсах команды разработки возможно руками создать золотой датасет таких же пар сообщение-JSON. Генерировать можно в обе стороны - разметчик получает JSON и пишет сообщение или наоборот, сам руками распаршивает в JSON.
